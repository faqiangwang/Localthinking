# GitHub Actions 构建错误收集指南

## 🔍 **查看构建状态的步骤**

### **第1步：访问Actions页面**

1. **打开浏览器，访问：**
   ```
   https://github.com/faqiangwang/Localthinking/actions
   ```

2. **查看最新的构建**
   - 你会看到 "Build and Release" 工作流
   - 状态会显示：
     - 🟡 蓝色圆点 - 运行中
     - ✅ 绿色勾号 - 成功
     - ❌ 红色叉号 - 失败
     - 🟠 橙色圆圈 - 排队中

---

## 🔴 **如何查看具体的错误信息**

### **方法1：展开失败的Job**

1. **点击失败的构建**
   - 在Actions页面找到最新的构建
   - 点击 "Build and Release"

2. **查看各个Job的状态**
   - 你会看到4个Job（每个平台一个）
   - 失败的Job会显示 ❌ 和红色背景

3. **点击失败的Job**
   - 点击红色的失败Job
   - 展开所有步骤

4. **找到失败的步骤**
   - 查找红色❌标记的步骤
   - 例如：
     - ❌ "Install dependencies (Ubuntu)"
     - ❌ "Build application"
     - ❌ "Upload artifacts"

5. **点击失败的步骤**
   - 展开后可以看到日志输出
   - 滚动查找 "Error:" 或 "Failed" 关键词

6. **复制错误信息**
   - 选中错误文本
   - Ctrl+C 复制
   - 粘贴给我

---

### **方法2：查看完整日志**

1. **在Job页面右上角**
   - 点击 "..." 菜单
   - 选择 "View logs" 或 "查看日志"

2. **搜索错误关键词**
   - 按 Ctrl+F 搜索：
     - `Error`
     - `Failed`
     - `error:`
     - `E:`

3. **复制相关错误**
   - 复制错误行及其前后几行
   - 这样我能看到上下文

## 🔍 **常见错误模式**

### **Linux构建常见错误**

#### **错误1：依赖安装失败**
```
E: Unable to locate package libwebkit2gtk-4.1-dev
```
**原因：** 包名或版本问题
**解决：** 更换包名或版本

#### **错误2：构建超时**
```
Error: Build timed out after 360 minutes
```
**原因：** 构建时间太长
**解决：** 增加超时时间或优化代码

#### **错误3：Rust编译错误**
```
error: linking with cc failed
```
**原因：** 依赖库问题
**解决：** 添加缺失的库

---

### **macOS构建常见错误**

#### **错误1：OpenSSL路径错误**
```
Could not find directory of OpenSSL
```
**原因：** OPENSSL环境变量未设置
**解决：** 已在配置中添加

#### **错误2：代码签名错误**
```
Error: Code signing failed
```
**原因：** 没有证书或证书配置错误
**解决：** 使用TAURI_BUNDLE_SKIP_SIGNING

#### **错误3：构建失败**
```
Command failed: cargo build
```
**原因：** Rust编译问题
**解决：** 检查依赖和环境
